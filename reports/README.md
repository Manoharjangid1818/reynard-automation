# Reynard Reports

This folder stores Playwright run output and custom history generated by `utils/customReporter.js`.

## Current Run Files

- `latest.json`  
  Full detailed result for the most recent run only. This is overwritten on every run and includes run metadata, git metadata when available, and the full `tests` array.

- `summary.json`  
  Append-only JSON array with one lightweight object per run:
  `{ runId, timestamp, totalTests, passed, failed, skipped, flaky, durationMs, gitBranch, gitCommit }`.

## History

- `history/<ISO-timestamp>_run.json`  
  Full detailed result for one specific run. The timestamp is filename-safe, so characters such as `:` and `.` are replaced with `-`.

- `history/by-test/<sanitized-test-name>.json`  
  Per-test history array. Each entry includes `{ runId, timestamp, status, durationMs }` so recurring failures and flaky behavior can be tracked over time.

## Test Result Fields

Each test entry captures the module/describe name, spec file name, test title, full title path, normalized status, duration, retry count, and failure details. Data-driven tests use the `tcTitle(tc)` format from `utils/testHelpers.js`, so titles like `[AUTH_TC_001] Valid login` are stored with `caseId`, `caseIndex`, `dataDrivenCaseId`, and `dataDrivenCaseIndex`.

## Legacy Outputs

The custom reporter also preserves the older outputs used by this project:

- `history/run_history.csv`
- `history/run_<timestamp>.csv`
- `json/<Module>_results.json`
- `json/last_run_summary.json`

Playwright's built-in reporters may also create:

- `html/`
- `json/results.json`
- `test-results/`
